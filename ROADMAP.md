# MiMi Wallpaper 项目路线图 (Roadmap)

这里记录着 MiMi Wallpaper 从零到一的进化历程，以及未来的发展方向规划。

---

## 🟢 当前版本：v0.1 (MVP 最小可行性产品) - `已上线`

**核心目标**：跑通底层的 AI 生成链路，确立品牌「极简猫派」的视觉基调，并完成公网部署。

**已完成功能**：
- [x] **核心功能**：实现“文生图”核心链路，基于 Google Gemini 大模型接口渲染壁纸。
- [x] **数据持久化**：接入 Supabase，实现生成图片云端直传和数据库（Url、Prompt）留痕。
- [x] **用户系统**：接入 Clerk 登录拦截系统，完美打通 Google 原账号及邮箱登录体系。
- [x] **UI/UX 设计**：采用黑白+琥珀强调色的定制猫咪视觉风格，支持细腻微交互。
- [x] **全链路部署**：Vercel 生产环境部署完毕，`mimiwallpaper.cn` 绑定并实现强制 HTTPS 访问。

## 🟢 规划版本：v0.3 (喵星表情包专区) - `已上线`

**核心目标**：横向扩展场景，不仅生成壁纸，还能一键生成微信/TG可用的各类搞怪喵咪聊天贴纸，增加趣味性和复用率。

**已完成功能**：
- [x] **独立路由/页面**：推出 `/emoji` 专区，配备全新的 Hero 引流版块。
- [x] **数据隔离**：重构底层接口逻辑，借助 `aspectRatio='emoji'` 的约束将表情包库和壁纸库完美解耦。
- [x] **特殊约束与优化**：对接 Gemini 大语言模型层注入专门用于表情包的约束（纯色白底、夸张情感风格）。

---

## 🟡 规划版本：v0.2 (高阶控制与拓展) - `计划开发中`

**核心目标**：赋予用户更强的画面控制力，引入「垫图提示」和「尺寸选择」，将产品从“玩具”升级为“工具”。

**待开发功能**：
- [ ] **多尺寸适配 (Aspect Ratio)**：
  - 允许在 Web 前端选择：1:1（默认）、16:9（桌面宽屏）、9:16（手机竖屏）。
- [ ] **艺术风格预设 (Style Options)**：
  - 增加常见动漫、机甲、写实、极致极简等内置提示词模版组合。
- [ ] **图生图能力 (Prompt-based I2I)**：
  - 增加参考图上传入口。
  - 使用 Gemini 多模态视觉模型（Vision）逆向解析上传垫图的色彩与构图风格。
  - 将「系统提取出的绝美垫图提示」与「用户口语化的要求」拼接，进而生成风格吻合的新图。
- [ ] **画廊体验优化**：
  - 加载状态（Loading）针对图生图专门设计新文案。
  - （可选）在画廊中引入 “仅查看我的创作” 过滤按钮。

---

## 🔵 未来展望：v1.0 (商业化基础) - `待定`

**核心目标**：跑通现金流商业模式，建立高阶使用门槛。

**蓝图设想**：
- [ ] 接入 LemonSqueezy 或 Stripe，提供“专业猫奴订阅包”或次卡。
- [ ] 建立账户点数体系（例如新用户送 5 点，每日签到送 2 点）。
- [ ] 加入 4K 超清分辨率无损放大功能 (Upscale)。
- [ ] 增加点赞、墙纸排行榜功能，鼓励用户裂变分享。
